Git Version Command

git config

  • git config --global user.name "Bilgehan Bezir" komutu kodları yazan kişinin ismi. Görmek istersen: git config user.name
  • git config --global user.email "bilgehanbezir741@gmail.com" komutu kodları yazan kişinin maili. Görmek istersen: git config user.mail

git status

  • git status komutu git'in klasördeki güncel durumunu gösterir.

git init

  • git init komutu klasörde daha önceden git kullanılmamışsa git init kullanarak uygun hale getirir.

git add

  • git add 'dosya_ismi' komutu belirtilen dosyayı indexliyor.
  • git add . komutu tüm dosyaları bir anda indexliyor.

git commit

  • git commit -m 'commit mesajı' dosyayı indexledikten sonra yaptığımız update veya değişiklikler hakkında yorum yapmamızı sağlıyor.

git log

  • git log komutu bir dosya indexlenip commitledikten sonra işlemler kaydediliyor ve yapılan son değişikliklerin listesini gösteriyor.

.gitignore

  • .gitignore bu defterin içine yazıcağım dosyalar gizlenecek git status yazdığın vakit görünmeyecek.

git branch

  • git branch komutu git branch komutunu yazarak projede var olan branchleri görüntüleyebiliyoruz.
  • git branch isim komutu 'isim' adında yeni bir branç açar lakin buna direkt geçmez manuel olarak geçmek lazım bunun için git switch kullanmalıyız.
  • git branch -d 'branch_isim' komutu belirttiğimiz branchı siler.

git switch

  • git switch komutu geçmek istediğin branch ismini yazarak o branche geçiyoruz.

git stash

  • Commit atmadan branch değiştirdik diyelim. O kaydetmediğimiz dosyaları bulmak için git stash komutunu kullanıyoruz.
  • git stash list komutu kaydetmediğimiz dosyaları listeliyor.

git merge

  • git merge ‘branch_isim’ komutu yan branchi ana branche bağlamaya yarıyor..

git restore

  • git restore 'dosya_ismi' komutu belirtilen dosyayı son comitteki eski haline getiriyor
  • git restore . tüm dosyaları bir anda son comitteki eski haline getiriyor.

git check out

  • git checkout 'gitmek istediğin commit id si' Commitler arası geçiş yapmamızı sağlıyor.
  • Commit id lerine git log yaparak bulabilirsiniz.

git reset --hard

  • git reset --hard 'gitmek istediğin commit id si'geçmişteki bi commite kalıcı olarak dönersin.

git revert

  • git revert 'gitmek istediğin commit id si' git reset --hard ın aksine önceki commite giderken sonraki commitleri silmiyor yeni bir commit açıp eski comiti son açtığımız commite çekiyoruz

git diff

  • git diff komutu iki farklı şekilde kullanılabilir:
  1. git diff 'branch1':'branch2' komutu branchler arasındaki farklılıkları görmemizi sağlar.
  1. git diff '1.commit id':'2.commit id' komutu commitler arasındaki farklılıkları gösterir.

git rebase master

  • git rebase master komutu lazım branclerle master merge yapılma sırasında gereksiz commitler oluşuyor ve bunlardan kurtulmak istersek bu komutu kullanmamız lazım lakin bunu kullanırken dikkat etmeliyiz.
Eğer projeyi bizden başka kullanan varsa bu işlemi yapmamalıyız aksi takdirde onların projesini bozabiliriz.
Branch içindeyken bu komutu kullanmamız
Projeyi henüz bizden başka geliştiren yoksa bu komutu kullanabiliriz.
Hem log temizlemek hemde tarihi tekrar yazmak için kullanıyoruz.

git remote

  • git remote komutu githubdaki repomuzun linkine bir atadığımız değişkenleri gösterir.
  • git remote add origin 'repo_url' repo linkine origin linkini atadık.

git branch -M main

  • git branch -M main main branşında olduğumuzdan emin oluyoruz.

git push

  • git push -u origin main origin e push ediyoruz.

git pull

  • git pull 'url' branch_ismi komutu githubdan proje çekmemizi sağlıyor.

git clone

  • git clone 'url' komutu github projesini klonlamamızı sağlıyor.
